# Platforms — 6 プラットフォームの性格と固有仕様

出典: /designing-for-ios ほか各 Designing for ページ、/layout（Specifications）、/spatial-layout /immersive-experiences /ornaments /eyes /focus-and-selection /remotes /complications /always-on /multitasking /windows /going-full-screen

共通原則: HIG は「プラットフォームに馴染む（feel at home）」ことを繰り返し要求する。同一アプリでもプラットフォームごとにナビゲーション部品・入力・情報密度を作り直す。単純移植をしない。

## iOS（iPhone）
- 性格: 中型高解像度ディスプレイ、片手/両手保持、縦横切替、移動中の利用
- タッチが主入力。ターゲット 44x44pt 最小。ナビゲーションはタブバー+階層 push が基本形
- safe area で Dynamic Island・ホームインジケータを回避。縦横両対応が基本、full-width ボタン回避、ステータスバーは原則表示（foundations.md 参照）
- 固有面: ホーム画面クイックアクション（4 件まで、補助経路）、ウィジェット、Live Activities、App Clips
- 主要画面寸法（抜粋、pt）: iPhone 16 Pro Max 440x956 / 16 Pro 402x874 / 16 393x852 / SE 4.7" 375x667。全表は原典 /layout

## iPadOS
- 性格: 大画面タッチ + キーボード/トラックパッド/Apple Pencil の複合入力。マルチタスク前提
- **ウィンドウは自由リサイズされる**（macOS 同等）。フルスクリーン設計を先に、compact への切替は最後まで遅らせる。1/2・1/3・1/4 分割サイズで必ずテスト（/multitasking, /layout）
- ナビゲーション: convertible tab bar（サイドバー⇔タブバー）が推奨選択肢。ビューが狭まるときは第三カラム（インスペクタ）から先に隠す
- ポインタのホバー状態、キーボードショートカットを提供する

## macOS
- 性格: 大画面・複数ディスプレイ・据置・多アプリ並行の生産性環境。精密なポインタ入力
- **メニューバーに全機能を**（structure-navigation.md 参照）。標準ショートカット尊重。ウィンドウのリサイズ・状態復元・フルスクリーン（/going-full-screen）に対応
- クリックターゲットはタッチより小さくできるが、アクセシビリティの 44pt 相当の配慮は残す。ウィンドウ下端に重要コントロールを置かない
- 固有部品: パネル、column/outline views、path controls、token fields、Dock メニュー、combo boxes、color/image wells

## tvOS
- 性格: 8 フィート以上離れた大画面を、リモコンの**フォーカス移動**だけで操作。ラウンジでの共有視聴
- **safe area: 上下 60pt、左右 80pt** インセット（コーパス検証済み）。画面サイズに自動適応しないため複数サイズで確認
- フォーカス時に要素が拡大する。**フォーカス可能要素の間隔**を確保（重なり防止）。グリッド仕様（unfocused 幅 / 水平間隔 40pt / 垂直間隔 100pt 最小）:

| 列数 | unfocused 幅 |
|---|---|
| 2 | 860pt |
| 3 | 560pt |
| 4 | 410pt |
| 5 | 320pt |
| 6 | 260pt |
| 7 | 217pt |
| 8 | 184pt |
| 9 | 160pt |

- テキスト既定 29pt / 最小 23pt。入力は最小限に（検索も音声・簡易グリッドで）。Top Shelf でハイライト提示

## visionOS
- 性格: 無限の 3D 空間。ウィンドウ / ボリューム / 空間全体（Full Space）の 3 段階の没入度を流動的に行き来する（/immersive-experiences）
- 入力: **視線で狙い、指でタップ**。インタラクティブ要素は**中心間 60pt 以上**離す（コーパス検証済み）。ホバーエフェクトが他要素を隠さない余白を取る
- ウィンドウ外のコントロールは **ornaments**（ウィンドウに追従して手前に浮く面）で。ウィンドウ境界の外（上の共有メニュー・下のウィンドウ操作）を侵食しない
- 重要コンテンツはウィンドウ中央付近に。z 軸に伸びすぎるコンテンツはクリップされる。ポイントは角度単位（距離でスケール）
- 色は実環境の壁・照明の反射で見え方が変わる。没入体験では人の安全と快適（モーション酔い）を最優先

## watchOS
- 性格: 手首の一瞥（グランス）で完結する数秒のインタラクション。腕を上げている間だけが勝負
- レイアウトは**画面端まで全幅**（ベゼルが余白の役割）。横並びコントロールは 2〜3 個まで（グリフボタン 3 / テキストボタン 2）
- Digital Crown がスクロール・値調整の主役。テキスト拡大は 140% まで対応
- 固有面: **Complications**（文字盤上の常時情報。複数ファミリー対応が喜ばれる）、Smart Stack ウィジェット、Always On（減光時表示 /always-on）、Workouts、autorotation（相手に見せる画面）
- 画面寸法（px）: 42mm=374x446, 46mm=416x496, Ultra 49mm=410x502, S7-9 41mm=352x430 / 45mm=396x484 ほか（全表は原典 /layout）

## クロスプラットフォーム設計の手順

1. 主要プラットフォームで情報設計を確定
2. 各プラットフォームでナビゲーション部品を置換（タブバー ⇄ サイドバー ⇄ メニューバー ⇄ フォーカスグリッド）
3. 入力手段の差（タッチ / ポインタ / Crown / 視線 / リモコン）ごとにターゲット寸法・ホバー・フォーカスを再設計
4. 情報密度を再配分（watch=1 画面 1 事実、TV=遠距離可読、Mac=高密度多ウィンドウ）
