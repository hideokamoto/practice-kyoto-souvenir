# Patterns（場面別の定石）と Inputs（入力手段）

出典: /launching /onboarding /loading /feedback /notifications /widgets /live-activities /settings /gestures /siri 他

## ライフサイクル別パターン

### 起動（/launching）
- **即座に起動する**（数秒待たせない）。iOS/iPadOS/tvOS では launch screen（最初の画面に似せた無内容の画面）で高速起動の印象を作る。ブランディング画面にしない（foundations.md の Branding 参照）
- 前回の状態を復元し、続きから使えるようにする。初回起動での大量の許可要求・設定要求を避ける

### オンボーディング（/onboarding）
- **理想は説明不要のアプリ**。オンボーディングが必要なら「速く・楽しく・スキップ可能」に
- 静的な説明より**操作しながら学ばせる**。単一の長いフローより、文脈に応じたヒント（tips）の分散提示を検討する
- 起動体験の一部ではない（launching 完了後に出す）

### ローディング（/loading, 2025-06 バックグラウンドダウンロード指針改訂）
- 最良のローディングは気づかれる前に終わる。**何かをすぐ見せる**（プレースホルダ・スケルトン → 実コンテンツに差替え）
- 待ち時間中も他の操作をさせる（バックグラウンドロード）。所要が既知なら determinate、未知なら indeterminate の進捗表示（components.md 参照）

### フィードバック（/feedback）
- 目的: 現在の状態 / 成否 / 警告 / 修正機会 の伝達。**情報の重大性と提示の強さを一致させる**（状態は受動的に、データ喪失の警告は割り込みで）
- 視覚・音・ハプティクスを組み合わせ、単一手段に依存しない

### 通知（/notifications, /managing-notifications）
- 事前の同意が必須。**一目で価値がわかる時宜を得た情報**だけを送る。マーケティング乱用は設定でオフにされ信頼を失う
- スタイル（バナー / バッジ / 通知センター）と緊急度レベルを適切に選ぶ。通知からのディープリンクは該当文脈に直行させる

### 設定（/settings）
- 「そのまま動く」が前提。設定は少ないほど良い。システム設定（外観・通知・プライバシー許可）と重複する項目をアプリ内に作らない
- アプリ内カスタム設定は体験全般に関わるもの（表示スタイル等）に限る。頻繁に切り替える項目は設定ではなくビュー内コントロールに置く

### 拡張面: Widgets / Live Activities
- **Widgets**（/widgets）: 少量の時宜情報をホーム画面/ロック画面/デスクトップ/Smart Stack に。開かず見られることが価値。サイズ別ファミリーごとに情報量を再設計する（縮小版スクリーンショットにしない）
- **Live Activities**（/live-activities）: 数時間スケールの進行中イベント（配達・試合・タイマー）の頻繁な更新を Lock Screen / Dynamic Island / StandBy / watchOS Smart Stack に表示。プッシュ通知の連打の代替

## Inputs（入力手段）

### ジェスチャ（/gestures）
- 全プラットフォーム共通の基本ジェスチャ（tap / swipe / drag 等）の**標準の意味を上書きしない**。システム全体のジェスチャ（ホームへ戻る等）と競合させない
- **ジェスチャを唯一の手段にしない**: 音声・キーボード・Switch Control 等の代替経路を必ず用意（accessibility 要求）
- 独自ジェスチャはゲーム等の没入体験でのみ慎重に導入し、学習手段を提供する

### プラットフォーム固有入力（詳細は platforms.md と原典）
- **キーボード / ポインタ**（iPadOS・macOS）: 標準ショートカットの尊重、ホバー状態の提供、フルキーボードアクセス
- **Digital Crown**（watchOS）: スクロール・値調整の主要手段。触覚フィードバックと併用
- **Eyes + ハンドジェスチャ**（visionOS）: 視線で狙い、指先タップで決定。凝視を強いる UI・小さすぎるターゲットを作らない（中心間 60pt）
- **リモコン / Focus**（tvOS）: フォーカス移動が唯一のナビゲーション。フォーカス可能要素の拡大表示と間隔を設計に織り込む
- **Siri / App Shortcuts**: 主要機能を音声・ショートカットから起動可能にする（/siri, /app-shortcuts）

### 触覚（Playing haptics ○ 原典参照）
- 意味のある瞬間だけに。システム標準のパターンを優先し、乱発しない
