# Web サイトへの翻訳指針

**注意: このファイルは HIG 原文ではない。** HIG はあくまで Apple プラットフォームのアプリ向け規範であり、Web への適用は本スキルによる翻訳（解釈）である。成果物で HIG を根拠として引く場合、この区別を明記すること。

## そのまま移せる原則（プラットフォーム非依存の設計原理）

| HIG の原則 | Web での実装 |
|---|---|
| 階層と読み順（Layout） | 見出し階層（h1-h6）、上・leading 側に最重要情報、RTL は CSS logical properties |
| タップターゲット 44pt | 44px 相当（WCAG 2.5.8 は最小 24px だが HIG 水準の 44px を推奨値とする） |
| コントラスト 4.5:1 / 3:1 | WCAG AA と同一基準（HIG 自体が WCAG を参照している）。そのまま適用 |
| Dynamic Type | rem ベースのタイポグラフィ + ユーザーのフォントサイズ設定・200% ズームで崩れないこと |
| ライト/ダーク両対応 | `prefers-color-scheme` + セマンティックなカラートークン設計 |
| Reduce Motion への応答 | `prefers-reduced-motion` で装飾モーションを無効化 |
| 色だけに依存しない | 同一（WCAG 1.4.1） |
| 段階的開示 | アコーディオン・「もっと見る」・スケルトンでの逐次表示 |
| モーダルの 4 条件（Modality） | ダイアログ（`<dialog>`）は同じ条件でのみ。多段モーダル禁止、フォーカストラップと Escape 対応 |
| アラート乱用禁止 | 確認ダイアログの最小化、取り消し可能操作には Undo（トースト+Undo）を優先 |
| 検索の一次化・サジェスト | 同一の設計判断 |
| ローディング: まず何か見せる | スケルトンスクリーン、ストリーミング/遅延読み込み |
| 起動即応・状態復元 | 初期表示速度（LCP）、フォーム・スクロール位置の復元 |
| ブランドはコンテンツに譲る | スプラッシュ的なイントロ演出・過剰なロゴ表示を避ける |
| プライバシー: 必要時に最小限 | 権限（位置・通知）は文脈のあるタイミングで要求。Cookie 同意も同様の思想で設計 |

## 移してはいけないもの

- **San Francisco / New York フォント**: 許諾が Apple プラットフォーム限定。Web ではシステムフォントスタック（`system-ui` 等。Apple デバイス上では結果的に SF が使われる）か別のフォントを使う
- **SF Symbols**: 同じく規約上 Apple プラットフォーム外での使用不可。Web では別のアイコンセットを使う
- **Liquid Glass の模倣**: 見た目の模倣（過剰な backdrop-filter）はパフォーマンスとアクセシビリティ（Reduce Transparency 相当の配慮が必要）のコストが高い。「コントロール層とコンテンツ層の分離」という構造原理だけを持ち込む
- **タブバー等の OS 部品の見た目コピー**: Web にはWeb の慣習（ヘッダナビ・ハンバーガー・フッタ）がある。「トップレベル区分の常時可視化」「現在地の明示」という機能要件に翻訳する
- **44pt 等の数値の盲信**: Web は閲覧距離・デバイスが多様。数値は出発点とし、実測（タップ成功率・ズーム時の崩れ）で検証する

## Web 固有で追加すべき観点（HIG が扱わないもの）

- セマンティック HTML と ARIA（HIG の VoiceOver 対応に相当する Web の手段）
- キーボードナビゲーション（フォーカスリングを消さない）
- レスポンシブブレークポイント（HIG のサイズクラスに相当する自前設計）
- パフォーマンスバジェット（Core Web Vitals）
- SEO・OGP 等のメタデータ
